When evaluating whether a practice makes sense in this project, base your reasoning on observed data, not general principles.

**Why:** Assessed `/self-analysis` as "selective, not after every PR" — because in "typical projects" friction is rare. In this project, 3 bots (Copilot, Gemini, Codex) generate 10-30+ comments on EVERY PR. User had to correct 3 points.

**How to apply:** Before saying "worth it / not worth it" — check data from the current session: how many review loop iterations, how many comments, how many pushes. If user asks for a practice evaluation — answer from this project's perspective, not abstractly.
