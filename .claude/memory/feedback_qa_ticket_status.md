---
name: QA tickets start as To Do
description: New QA tickets in Jira should stay in To Do status, not In Review
type: feedback
---

Create QA tickets for the tester in **To Do** status — don't transition to In Review.

**Why:** In Review implies someone is already reviewing. The tester will move it themselves when they start testing.

**How to apply:** When creating QA tickets in Jira (<JIRA_PROJECT_KEY>), don't execute any transition — leave the default To Do status.
